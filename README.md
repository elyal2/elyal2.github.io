# LLM Demos: Single-Page HTML5 Applications

Este repositorio contiene una colección de demos públicas de mini-aplicaciones web (HTML5, CSS3 y JavaScript) diseñadas para ser ejecutadas directamente en el navegador como una sola página.

Todas estas aplicaciones han sido desarrolladas utilizando **vibe-coding** con LLMs (Modelos de Lenguaje de Gran Tamaño), priorizando la rapidez de prototipado y la funcionalidad directa.

## 🚀 Demos Disponibles

Puedes acceder a las demos directamente a través de GitHub Pages:

- **[IPTC & FIBO Classifier/NER](https://elyal2.github.io/iptc-fibo-classifier_ner.html)**: Una herramienta para clasificación de textos bajo estándares IPTC y FIBO, incluyendo extracción de entidades (NER).

## 🛠️ Cómo crear tu propio repositorio de demos

Si quieres publicar tus propias demos de vibe-coding de forma similar, aquí tienes una pequeña guía paso a paso:

### 1. Crear el Repositorio

Crea un nuevo repositorio en GitHub. Para que sea tu sitio principal, nómbralo siguiendo el patrón: `tu-usuario.github.io`. Si prefieres que sea un proyecto separado, puedes ponerle cualquier nombre (ej. `mis-demos-llm`).

### 2. Sube tus Archivos

Sube tus archivos `.html` directamente a la raíz (o en carpetas). Asegúrate de que sean "Single Page Apps" (todo el CSS y JS embebido o llamado desde CDNs) para que funcionen sin necesidad de un servidor de build (como Vite o Webpack).

### 3. Configurar GitHub Pages

1. Ve a la pestaña **Settings** (Configuración) de tu repositorio.
2. En la barra lateral izquierda, haz clic en **Pages**.
3. En la sección **Build and deployment**, asegúrate de que **Source** esté en "Deploy from a branch".
4. Selecciona la rama `main` (o `master`) y la carpeta `/ (root)`. Haz clic en **Save**.

### 4. Acceder a tus Demos

Una vez GitHub termine el despliegue, tus aplicaciones estarán disponibles en:
`https://tu-usuario.github.io/nombre-del-archivo.html`

## ✍️ Sobre Vibe-Coding

Estas herramientas son el resultado de iteraciones rápidas con IA. No buscan ser productos finales perfectos, sino demostraciones de lo que se puede lograr comunicando la "vibración" y el objetivo de una idea a un LLM para obtener código funcional al instante.
